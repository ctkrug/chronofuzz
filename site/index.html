Chrnofuzz

Paste a date function. Find out exactly where it breaks.

Your date code has a landmine in it. Chronofuzz shows you exactly where.

Paste a JavaScript or Python date-handling function and run it against a curated, research-backed battery of real date/time failure modes like DST spring-forward gaps, impossible leap days, the Year 2038 rollover, and leap seconds. Hit run and get an immediate, specific diagnosis: which landmine broke, and the exact wrong value your function returned.

What a broken line looks like

FAIL DST spring-forward: the clock skips 2:00–3:00 AM dst

Accepted a time that never existed

returned2023-03-12T02:30:00.000Z

2:30 AM never occurred on this date in this zone. Clocks jumped from 1:59:59 to 3:00:00, so a correct implementation should reject the input or normalize it forward to 3:30 AM EDT (07:30 UTC), not accept a wall-clock time that never existed.

How it works

  1. 1

    Paste your function

    JavaScript or Python, toggle between them. Each language starts pre-filled with a naive sample so you can see it break in seconds.

  2. 2

    Run the battery

    Your function runs once per landmine in an isolated sandbox (a fresh Web Worker per run for JavaScript, Pyodide for Python) with no network access and no access to the host page.

  3. 3

    Read the verdict

    Every landmine gets a real pass, fail, or ambiguous verdict from an automated evaluator, never a raw dump for you to eyeball. Failing rows show the wrong value inline, with a diff against what a correct implementation returns.

Six categories, one battery

Every run checks your function against every real-world failure mode below: no timezone database lookups, no remembering which years are leap years.

  • DST3
  • Leap day4
  • Leap second3
  • Epoch boundary3
